## 手作業フローとの整合（mac-mini-bot が Vercel に渡す Git URL）— **必須・省略禁止**

**このブロックがプロンプトに含まれている限り、GitHub への push を完了したあとのあなたの返答には、必ず `BOT_DEPLOY_GITHUB_URL:` 行を含めること。** 含めないこと、別メッセージや追伸で後から送ること、「URL はダッシュボードで確認」などで省略すること、**いっさい禁止**。1文字も欠けたら下流の自動処理が失敗し案件が止まる。

リファクタ結果のソースを **あなたの環境で GitHub に push** したうえで、返答全文の **最後の非空行を、次の1行だけ** にすること。**この1行の後に文字・空行・見出し・箇条書きを一切書かない**（工程テスト・本番のパースがここだけを読む）。**特に `BOT_DEPLOY_GITHUB_URL:` の行の直後に「タスクが完了しました」「以上」等を書くと、古いパーサでは URL が読めず案件が失敗するため、報告文は必ずその行より前に書き、URL 行で返答を終えること。**

`BOT_DEPLOY_GITHUB_URL: https://github.com/オーナー/リポジトリ.git`

- 上記1行は **フェンス外**。**それより前**は従来どおり「1ファイル=1フェンス」のマークダウンのみ（パース規則は変えない）。
- push したコミットの内容は、直前のフェンス群と **同一** にすること（ボットはローカルにも同じフェンスを適用し `npm build` する）。
- **禁止**: 最終行を「GitHubプッシュ完了」・コミットメッセージの列挙・作業サマリー・感嘆文にすること。**報告はすべて BOT_DEPLOY 行の直前まで**に書く。最終行は **必ず** `BOT_DEPLOY_GITHUB_URL:` で始まり、**有効な `https://github.com/...` の clone 用 URL を続けた1行のみ**（`.git` 付き推奨）。
- **再掲（必須）**: 返答を終える直前に、上記1行を**必ず**出力すること。これが無い返答は要件未達とみなす。
