以下の手順を順に実行せよ。停止・質問・サボりは禁止。無限ループを回避し正確に実装すること。大規模な作業になってしまう場合は、同時に作業するのではなく、一つずつ確実に実装するようにしてください。

【表現】ソース・コミットメッセージ・作業メモに **装飾用の Unicode 絵文字を含めない**。ユーザー向け UI 文言にも **原稿に無い絵文字を追加しない**。

【手順1: リポジトリ作成】
propagate-webcreationのワークスペースにリポジトリを新たに作成してください。
- **リポジトリ名**は次の文字列と**完全一致**させること: `{{MANUS_REPO_NAME}}`
  （フォーマットは「レコード番号-ASCII部分」。GitHub はリポジトリ名に日本語を受け付けないため、パートナー名の英数字部分のみを使用。ASCII 部分が無い場合はレコード番号のみ。**この文字列を1文字も変えないこと**）
- **ディスクリプション**には次の1行を**そのまま**入力すること: `{{MANUS_REPO_DESCRIPTION}}`
  （「レコード番号 パートナー名」形式。制作スプレッドシートの値そのまま。工程テスト・本番を問わずこの形式）
privateのままで、テンプレートリポジトリとしてpropagate-webcreation/DefaultSettingを使用してリポジトリを作成してください。
※もし、作成しようとしたリポジトリ名が既にあるなら直ちに作業を終了してください。

【手順2: リファクタリング】
作成したリポジトリをクローンして、以下に記載された「リファクタリング元のソースコード」を以下の「リファクタリング指示書」に従ってリファクタリングを実装してください。
元のソースコードは単一ファイルですが、複数ページ（HomePage, AestheticPage, ImplantPage等）を内包している可能性があるので確認してください。シングルページの場合もあります。
各ページをNext.jsのルート（/, /aesthetic, /implant等）として分離し、さらに各ページ内のセクションを個別のコンポーネントファイルに分割してください。
各page.tsxでは、そのページ用のセクションコンポーネントをimportして配置するだけにしてください。

完了前の必須アクション
【CSS実装時の必須ルール】
globals.cssやCSSファイルを作成・編集する際は、以下のルールを厳守してください：
1. **@importルールの配置:** 必ず@importルールをファイルの先頭（コメントの後）に配置すること
2. **順序の厳守:** :root変数、@theme、その他のCSSルールは必ず@importの後に記述すること
3. **外部フォントの読み込み:** Google Fontsなどの外部フォントはCSS @importではなく、layout.tsxのHTML <link>タグで読み込むこと
4. **正しいCSS順序テンプレート:**
```css
/* コメント */
@import "tailwindcss";
:root { /* CSS変数 */ }
@theme { /* Tailwind設定 */ }
/* その他のCSS */
```
実装完了後、指示書の「追加の絶対制約」に違反していないか必ず自己レビューを行い、修正してから次の手順に進んでください。

【手順3: 画像生成と実装】
コード内の`ImagePlaceholder`等のtext/subText、プレースホルダーやサイト全体のトーンを分析し、nanobananaProで画像を生成・実装する。

■人物描写ルール
* **基本ルール:** 必ず「日本人（Japanese）」を指定してください。
* プロンプト例: `Japanese professional`, `Japanese woman`, `Japanese office environment`

■プロセス
1. **分析とプロンプト作成**
**画像プレースホルダーの特定:**
- ImagePlaceholder系コンポーネント（名称問わず）
- text/imgText等のprops、aspect-ratio/heightClass等の画像領域
- コンポーネント内の動的生成（map/loop/配列データ）、子コンポーネント内も含む
**抽出手順:** 元のソースコード全体から上記を全て抽出し、漏れがないことを確認。
以下のJSON形式で出力。promptに"no text"必須。**全プレースホルダー分析必須。**
```json
[
  {
    "id": "ファイル名",
    "prompt": "(英語プロンプト。人物描写のルールを適用すること。高品質な呪文も含む)",
    "negative_prompt": "low quality, text, watermark, bad anatomy",
    "aspect_ratio": "(16:9 または 4:3 など)",
    "reason": "選定理由"
  }
]
```

2. **画像生成 (Step 1)**
JSONに基づきnanobananaProで生成。文字混入厳禁。必ず全てのプロンプトで画像を作成すること。
画像保存: 生成した画像は /public/images/ ディレクトリに保存してください（ディレクトリがない場合は作成すること）。
【必須】文字の完全除外: 生成する画像に文字、テキスト、レター、数字、記号などの書かれた文字を一切含めないこと
## 画像生成時の必須チェックリスト
- [ ] 文字・テキストを一切含めない
- [ ] 人物は必ず「Japanese」を指定

3. **実装 (Step 2)**
- 保存先: `/public/images/` (無ければ作成)
- パス置換: ソース内の外部URL等をローカルパス(`/images/xxx`)に置換。`src`に`/public`は含めない。
- コンポーネント: `next/image`から`Image`をインポートして使用。ローカル画像をパス文字列で指定する場合、<Image /> コンポーネントには必ず width と height プロパティを指定するか、fill プロパティ（親要素に relative 必須）を使用してください。これを守らないとビルドエラーになります。
既存のクラス名（Tailwind CSS）は維持し、デザインが崩れないようにしてください。
**画像実装の完全性確認（必須）:**
1. 元のソースコードで画像が表示されていた箇所を全てリストアップ
2. 生成した画像の数と、元のソースコードの画像表示箇所の数が一致することを確認
3. 不一致の場合は、漏れている箇所を特定し、追加で画像生成を実施
4. 実装後、全ページの全セクションで画像が正しく表示されていることを確認
5. 画像プレースホルダーが残っている箇所がないことを確認

4. **検証とプッシュ (Step 3)**
- `npm run build`を実行。
- エラー時は人間に聞かずログを読み、成功するまで「修正→ビルド」を繰り返す(自己修復)。
- ビルド完全成功後にプッシュする。

モデルは **Manus 1.6** を使用すること（手作業マニュアルどおり）。
