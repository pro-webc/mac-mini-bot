## 実行姿勢
停止・質問・サボりは禁止。大きい場合は同時多発ではなく**一段階ずつ**正確に実装する。

## スコープ（本チャットでは行わない）
- **ボットの後続バッチ**による画像 API（パイプラインが TSX を機械置換する処理）。画像は**本チャットの返答（フェンス）だけ**で完結させる。

## スコープ（本チャットで行う）
- `===== BEGIN_CANVAS_SOURCE =====` 〜 `===== END_CANVAS_SOURCE =====` 内の**単一ファイル**を、続く**リファクタ指示書**に従い Next.js App Router + Tailwind 向けに**複数ファイル化**し、**テキストのみ**で返す。
- **サイトのソースの正本はあなた（Gemini）の返答（フェンス内）だけ**とする。ローカルの空テンプレやデモ用 Next 雛形に「載せる」前提は置かない。必要なファイルはすべてここで出力する。
- BASIC LP / BASIC-CP / STANDARD-CP / ADVANCE-CP は**同一のリファクタ指示書**（直後の本文）と、このメッセージ前半までの**共通スコープ**を使う。プラン差分は各プラン専用の intro（1段落）のみ。
- `npm run build` は実行不可。返答前に型・import・Server/Client・Hydration・**page.tsx 薄さ**・**globals.css 順序**を自己レビューする。

## 出力形式
- 変更・追加する**全ファイル**を、**1ファイル＝1フェンス**で列挙する。**直後のリファクタ指示書の「※ 自動適用ボットとの整合」**に書いたとおり、開始直後（言語タグの次行、または ` ```app/... ` 形式）に **`app/` / `components/` 等ボットが許可する相対パス**を置く。これを外すとボットがファイルを0件抽出し失敗する。
- `app/**/page.tsx` は **import + `<Component />` のみ**（className・ラッパー要素禁止。詳細は指示書 §3）。

## 画像（最終リファクタでここまで実装）
- マニュアル段の **`ImagePlaceholder` はすべてやめ、`next/image` の `Image` に置換**する。`src` は **`/images/...`（= `public/images/...`）** のローカルのみ。
- **同一返答内**で、必要な **`public/images/*.svg` を追加フェンスとして出力**する（SVG はテキスト。ヒーロー・サムネは viewBox・図形・グラデで簡潔に表現してよい）。外部 URL・Unsplash・ストック CDN は禁止。
- ラスタがどうしても必要なときも **まず SVG で近似**。`remotePatterns` に頼る外部ホストは使わない。
