## 分析手順

1. **ステータス判定** — Manus が以下のどの段階まで完了したかを応答から読み取る
   - リポジトリ作成
   - リファクタリング（コード変換・画像生成）
   - npm build 成功
   - GitHub push 完了

2. **GitHub URL 抽出** — `BOT_DEPLOY_GITHUB_URL:` 行、または本文中の `https://github.com/...` URL を見つけて正規形 `https://github.com/{owner}/{repo}.git` にする

3. **失敗時のリカバリ判断** — Manus が失敗・途中停止した場合、応答テキストから原因を特定し、回復策を選択する:
   - **リポジトリが既に存在する** → `USE_EXISTING_REPO`（既存リポに有効なコードがある可能性。URL を構築して返す）
   - **ビルドは成功したが push 失敗** → `LOCAL_PUSH`（コードは GitHub 上にない。ボットがローカルから push する）
   - **リファクタ途中で停止** → `RETRY_MANUS`（再実行の余地あり）
   - **全工程完了** → `PROCEED`
