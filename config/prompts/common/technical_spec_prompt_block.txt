【ソースの正本（全プラン共通・最優先）】
- 各ページ・レイアウト・ヘッダー/フッター・セクション・スタイルは **すべて本パイプライン上の TEXT_LLM がコードとして出力する**。リポジトリ内のデモ用サンプルや別案件向け素片に依存・流用しない。ヒアリング・page_structure・仕様（design_spec 等）のみを根拠に、**あなた（TEXT_LLM）の返答がサイト完成形の唯一の正本**である。

【技術要件（全プラン共通・必須遵守）】
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成すること（1セクション=1コンポーネントを原則）
- CSSの@import（Google Fonts等）は、CSSファイルまたは<style>の最上部に配置。:root・セレクタ・プロパティ等の他ルールより前。守らないとビルドエラー
- 図解・表は画像ではなく、マークアップ＋Tailwindでコーディング生成
- アイコン: 基本は Lucide React（https://lucide.dev/icons で実在する名前のみ import）。`Pipe` 等の存在しない名前は使わない（配管イメージは `Droplets` / `Wrench` / `Cable` 等）。SNSは Simple Icons。アイコンを画像プレースホルダにしない
- Unsplash・外部ストック URL 禁止。**ヒーロー（メインビジュアル）には必ず画像を用いる**（`next/image` + `public/images/`。**画像なしの単色／CSS模様のみのヒーローは禁止**。オーバーレイのグラデーションは画像の上に重ねてよい）。**ビジュアル（最低 3 意図箇所・ヒーローに必ず 1）は、マニュアル後の最終リファクタ（Manus）で実装**する：`next/image` の `Image` と **`public/images/` 以下の静的ファイル**（同一返答のフェンスで `.svg` 等を出力可。マニュアル段の `ImagePlaceholder` はここで置換）。ボットの別工程による一括画像 API はない。画像上のテキストはオーバーレイで表示
- **next/image のパスと属性（表示欠け・ビルドエラー防止）**: `Image` の `src` は **`/images/...` のように public ルート直下の相対パス**（先頭 `/`）。**`src` に `/public/` や `public/` を含めない**（`public/images/foo.png` は URL 上は `/images/foo.png`）。参照先ファイルは**フェンス出力またはリポジトリに実在する**こと。**存在しないパスのみを src に書いて終わらせない**。`width` と `height` を数値で渡すか、`fill` を使うときは親に **`relative` と高さまたは `aspect-*` で表示領域を確定**させる。リモート URL を使う場合は `next.config` の `images.remotePatterns`（等）が必須で未設定だとビルド・最適化で失敗し得るため、**本案件ではローカル静的ファイルを既定**とする
- 所在地: Google Maps 埋め込み。マップエリアに画像・プレースホルダ禁止。対応エリア用の画像・イラストは禁止。所在地が明確ならピン表示
- **影・立体感（緩和済み）**: カード・料金パネル・特徴ブロック・モーダル等の**コンテナ**には、Tailwind の **`shadow-sm` / `shadow-md`** または **`ring-1 ring-black/5`（`ring-*`）** を使って階層を付けてよい。区切りは**境界線・余白・タイポ階層と併用**する。**`shadow-2xl` 以上の濃いドロップシャドウの乱発・ネオン風の glow・テキスト本体に直接当てる読みにくい影**は避ける

【スタイリング要件（全プラン共通・高額制作向け）】
- **design_spec の color_scheme / tokens を最優先**し、仕様に無い派手な色の追加・勝手な全面ベタ塗りは禁止。`body` / `html` の背景は **白・stone/zinc/neutral の 50〜100 系など低彩度**を既定とし、ブランドの強い色は**ヘッダー帯・CTA・リンク・アクセント要素**に限定する
- テキスト色は **コントラスト確保のため要素に明示**するが、**色の種類は design_spec で定めた palette に収める**（本文・弱め本文・見出し・primary上で足りる構成に）。「要素ごとに別々の任意 hex を乱発」しない
- ボタン・リンクは hover / active / focus-visible / disabled を **primary 系の明度差**で揃え、**虹色の複数CTA色**にしない
- タイポグラフィは仕様の typography に沿い、長文は `max-w-prose` 相当の可読幅を確保。見出し階層（h1 はページ1つ）を乱さない。**メインビジュアル上の H1・キャッチは design_spec のタイトル／見出しサイズどおり本文より大きく**し、背景とのコントラスト（オーバーレイや背景グラデーション）で必ず読めるようにする
- セクション間の **縦リズムを揃える**（例: 主要セクションは同一の py スケール）。詰め込みすぎない
- Tailwind の任意値 `bg-[#..]` は **design_spec で定義した色の反映に限定**し、実装者の即興で飽和色を増やさない

【没入感・世界観・現代的UI（全プラン共通）】
- **一貫した世界観**: design_spec のコンセプトに沿い、**色温・角丸・線種・余白スケール・見出しの語り口**をサイト全体で統一する。セクションがバラバラのテンプレ寄せにならないよう、**視覚的メタファー**（例: 信頼＝整然としたグリッドと十分な余白、温かみ＝柔らかい面分割と段階的なタイポ）をコード上でも維持する
- **引き込み・視線誘導**: ファーストビューで**価値と主CTA**が一瞬で分かること。スクロールに沿って**認知→納得→行動**の順になるよう、見出し・リード・図解・CTA の**リズム**を単調な縦積みだけにしない（背景トーンの交互、フル幅とカード幅の切り替え、`max-w-*` による読みの芯など）。**メインの見出し・本文・料金表・フォームを大きく傾けて配置するレイアウトは避ける**。バッジ・ラベル・アイコン装飾など**軽い角度（目安 ±8deg 以内）**は可
- **カード・コンポーネントの仕上げ**: 料金・特徴・事例・ステップ等のカードは **`rounded-*` の統一**、`p-*` / `gap-*` の均一、`flex flex-col` + **`justify-between`** でCTAやフッターを揃える等、**プロダクト品質のカードUI**を意識する。`group` / `group-hover:` で**画像・境界線・アイコンの連動**を付けてもよい
- **現代的UI・インタラクション**（上記の影ルールと併用）: レスポンシブ段組、`@container`、`scroll-mt-*`、必要なら **`sticky` サブナビ／目次**。ホバー・フォーカスは **`transition-colors` / `transition-transform` / `duration-200`〜`300`**。カードは **`hover:-translate-y-0.5`〜`1` + `hover:shadow-md`** 等の**軽いリフト**、`active:scale-[0.98]` 程度の**押下フィードバック**を付けてよい。スクロール入場は **`motion-safe:` 内**で **`opacity`・軽い `translate-y`・控えめな `scale`**。複数カードは **`delay-75` 段階の stagger**（最大 ~300ms 程度のずらし）まで可。**自動再生・無限ループ・点滅・大きなバウンス・視認性を損なうパララックス過多**は禁止
- **`prefers-reduced-motion: reduce`**: 装飾的な動きは実質オフ（`motion-reduce:transition-none` 等）。アクセシビリティを損なう演出は禁止
- **長文コンテンツの読みやすさ**: ボリュームのある本文は **`max-w-prose` 相当＋十分な行間・段落間**で「読む気になる」版面にする。情報は削らず**版面と階層**で没入感を補う（コピー正本の省略は別節の禁止どおり）

【UX・読みやすさ・操作（全プラン共通・実装で守る）】
- **タップ領域**: 主要なリンク・ボタンは **min-h-[44px] min-w-[44px]** を目安に確保（モバイル）
- **フォーム**: 各入力に **視覚的に紐づく label**（`htmlFor` / `id`）。`placeholder` は補助のみ。**必須**はラベル側で分かるように。`inputmode` / `autocomplete` は仕様の function_spec に従う
- **フォーカス**: キーボード操作で辿れること。`focus-visible:outline` または同等でフォーカスリングを消さない
- **動き**: 没入感用のモーションは **【没入感・世界観・現代的UI】** に従う。過剰・自動再生・点滅は禁止。`prefers-reduced-motion:reduce` では動きを抑える（`motion-safe:` / `motion-reduce:` の併用。装飾入場は `motion-reduce:opacity-100 motion-reduce:translate-y-0` 等で実質オフ）
- **ランドマーク**: `<main>` を各ページに一つ。`nav` に `aria-label`。見出しは階層を飛ばさない（h1→h2→h3）
- **情報の優先度**: モバイルで横スクロールを起こさない。長い表は `overflow-x-auto` またはカード化を検討（仕様に表がある場合）

【法的・ポリシー情報の表示（全プラン共通）】
- プライバシーポリシー・利用規約などは **契約で定めたページ数には含めない**。**新規の `app/**/page.tsx`（例: `app/privacy/page.tsx`）を増やして独立ルートにしない**。
- 全文が必要な場合は **既存のいずれかのページ上**で `Dialog` / `Sheet` / `Drawer` / `<details>` 等により表示する。フッターからは **同一ページ内の UI** で開く（**/privacy 等への `Link` 新設は禁止**）。

【レイアウト・レスポンシブ（全プラン共通）】
- **モバイルファースト**: 狭い幅で破綻しないことを先に満たし、`sm:` `md:` 等で段階的に余白・段組・タイポを足す
- **幅の制御**: 本文ブロックは `max-w-prose` または仕様のコンテナ幅に合わせ、`w-screen` や固定 `px` の乱用で viewport をはみ出さない。必要ならラッパーに `overflow-x-hidden` を検討（アニメ・横長要素で横スクロールが出る場合）
- **画像と比率**: `next/image` は親幅に追従させ、レイアウトシフトを避ける（`width`/`height` または `fill`＋`aspect-*` 等、仕様に沿った指定）

【パフォーマンス・LCP（全プラン共通）】
- **ファーストビュー**: ヒーロー等 LCP になり得る画像には `priority`（または同等の最適化）を付与。装飾・下層の画像は遅延読み込みを前提に `sizes` を適切に指定し、不要に巨大な元画像参照を避ける

【CTA・視覚的階層（全プラン共通）】
- **主アクション**: 1 ビュー（ファーストスクリーン）あたり「最も取ってほしい行動」を **primary CTA 1 系統**に絞り、副次は `outline` / `ghost` 等で一段下げる（虹色の複数 primary は禁止、既出ルールと併せて守る）
- **冗長な重複 CTA**: 同一文言のボタンを短いスクロール内に何度も並べず、セクションの役割に応じて表現を変えるか回数を抑える

【コピー・情報密度・完成度（全プラン共通・実装で最優先）】
- **構成案・ワイヤ・【テキスト情報】・手順3/4の確定文案は正本**。実装で**見出し・リード・本文・表セル・FAQ Q/A・ボタン文言を省略・要約・段落マージで減らさない**（レイアウト調整・「読みやすさ」・余白確保を理由にした短文化は禁止）
- **セクション薄化禁止**: 見出し＋キャッチ1行だけで終えない。主要ブロックには**リード1文（誰の・何が解決するか）**か同等の導入を置き、その下に**根拠・条件・手順・比較・事例型の具体**のいずれかを必ず載せる。**世界観に沿った十分な文量**を維持し、短文化でテンションだけ上げて中身を落とさない
- **ダミー禁止**: Lorem ipsum・無意味な英文の羅列・「サンプルテキスト」「ダミー」のみのブロックをユーザー向けUIに出さない。未確定は**短い日本語プレースホルダ**（例:「（料金は要確認）」）に留める
- **FAQ**: 各回答は**結論＋補足で原則2文以上**（Yes/No 一言完結禁止）。論点は業種で想定される**不安・障壁・手続き**をできる限り覆う
- **表・料金・フロー**: 列・行・ステップを**仕様どおりすべて表示**。「続きはお問い合わせ」のみで中身を空にしない（数値の創作は下記信頼性ルールに従う）
- **マイクロコピー**: フォーム送信前に**期待値**（返信目安・必須の意味）を1文以上。エラー時は**次に何をすればよいか**が分かる短文を用意（仕様にある場合）

【コンテンツの信頼性（全プラン共通）】
- **数値・実績・料金・保証・受賞**: ヒアリング・仕様に**無い具体数値や実在固有名の創作は禁止**。不足時は「要ヒアリング」「プレースホルダ」として**後工程で差し替え可能な文言**に留め、架空の実績を事実として書かない
- **法的・医療等の高リスク表現**: クライアント業種に応じ、断定的効果表現や誇大広告になり得るコピーを避け、**依頼内容の範囲で**中立・正確な表現に寄せる

【表現・文字面（絵文字等・全プラン共通・Override禁止）】
- **絶対禁止**: Unicode 絵文字（例: 顔・手・記号の絵文字）、絵文字ピッカー相当の記号を、サイトの**見出し・本文・リスト・表セル・ボタン・FAQ・フォームラベル・ナビ・プレースホルダ文言・装飾的な箇条書き**に含めない。装飾目的で**顔文字を絵文字で表現**するのも禁止。
- **属性・メタ**: `aria-label`・`alt`・`title` も**装飾用の絵文字を付けない**（事実・短い説明のみ）。
- **コード・リポジトリ**: ソース内のユーザー向け文字列、**コミットメッセージ**、README の説明文に**装飾用の絵文字を入れない**。
- **あなた（モデル）の返答**: チャット上の説明・確認文にも**絵文字を使わない**（プレーンテキストのみ）。
- **例外**: ヒアリング・原稿に**明示的に含まれる**絵文字だけ**そのまま転記**可。原稿に無い限り**新規追加は禁止**。
- **代替**: 強調は**タイポの階層・余白・配色・Lucide 等の意味のあるアイコン**で行う（アイコンを絵文字代わりに乱用しない）。
