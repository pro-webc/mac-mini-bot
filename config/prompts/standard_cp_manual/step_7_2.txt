**【手順.7-2】**

以下のサイト構成に含まれる**ユーザー向けの情報（見出し・本文・リード・表・ボタン・FAQ 等）を欠落・改悪なく**反映した TOP ページを作成してください。まずは TOP のみ作成してください。

【デザインの自主性（構成案との役割分担）】
- **構成案は「何を伝えるか・どの順で・どのボタンでどこへ」の正本**とする。掲載する**文言・数値・FAQ・表の内容**は構成案と手順 6 に従い欠落させない。
- **見た目のデザイン（レイアウト、セクション内の組み立て、カード/グリッド/余白の付け方、ヒーローのビジュアル構成、装飾の細部）は、手順 6 のデザイン指示書と、プロとしての判断で自律的に決める**こと。構成案の【画像・配置の指示】は**雰囲気・意図の参考**に留め、**文言どおりの配置やワイヤ再現を強制しない**。より読みやすく信頼感のあるコーポレートサイトになるよう**自分でデザインしてよい**。
- 手順 6 の**配色・フォントサイズ範囲・角丸・禁止事項**は必ず守る。

【原稿の厚み（必須）】
- **レイアウト調整のために文章を短くしない**。リード・本文・箇条・表セル・FAQ の**文量・項目数を要約・省略・マージで減らすことは禁止**。
- **見出しのみのセクションを新設しない**。構成案【テキスト情報】に含まれる**すべての文・行**を UI に載せる。
- 画像エリアを増やす・余白を取ることが理由で**段落やリスト項目を削除しない**。

【表示するテキストと「裏方メモ」の分離（必須）】
- サイト構成にある「【画像・配置の指示】」の本文、「メインビジュアル」「Why」などの**セクション見出しラベル**、制作向けの説明文は、**公開ページ上には一切表示しない**。
- 画面に出してよいのは「【テキスト情報】」ブロック内の**見出し・リード・本文・箇条書き・表のセル・ボタン文言・FAQ の Q/A**のみ。それ以外を段落として並べることは禁止。
- 「100%反映」とは上記のユーザー向け情報が**欠落しないこと**を指す。**レイアウト指示文をそのままユーザー向け段落として貼ることは禁止**（情報はデザイン・レイアウトに翻訳して反映する）。

【画像プレースホルダー】
- 画像エリアは**控えめな高さ**の枠と、**短いプレースホルダ文言**（例:「画像」または 18 字以内の要約）のみ。構成案の長い撮影指示は `{/* ... */}` コメントまたは `aria-label` に**1 文で要約**してよい。接頭辞「【画像プレースホルダー】」などをユーザーに見せない。
- 構成案で「画像の上に載せる」とある**キャッチ・タイトル・CTA**は、必ず**前面（オーバーレイ）**にタイポとして表示する。背景枠の内側に指示文を書かない。

【UI / デザイン品質（プロ品質）】
- **レイアウト**: セクション間の余白、カード内の `gap`、グリッドの揃えを統一し、要素の詰め込みを避ける。本文は `max-w-prose` または同等の `max-w-*` で可読幅を確保。
- **メインビジュアル（ヒーロー）**: **ヒーローには必ず画像を用いる**（`next/image` またはプレースホルダ枠。単色のみで画像を省略しない）。H1・キャッチは**手順 6 の「タイトル」px**で**本文より明確に大きく**実装。画像の上は**暗オーバーレイ**または**背景専用グラデーション**で文字を読ませ、キャッチ/H1/サブコピー/主CTA に十分な縦 `gap`。主CTA はヒーロー内で最も目立つ塗りボタン（`shadow-*` は使わない）。
- **タイポグラフィ**: 手順 6 のフォントサイズ規則を遵守しつつ、見出し 〉 リード 〉 本文の階層をウェイトと余白で明確に。同じ階層の見出しサイズは統一。**H1・ヒーローキャッチは本文帯（16〜18px / 12〜14px）の外の大きさを必ず適用**。
- **インタラクション**: リンク・ボタンに hover 差分。`focus-visible:outline` 等でキーボード操作時も視認できるようにする。
- **CTA**: 主アクションは最も目立つ塗りボタン。ナビ内の CTA は他 CTA と同系のボタン見た目にそろえる。
- **レスポンシブ**: 横スクロールを出さない。モバイルでタップしやすい余白を確保。
- **装飾**: 手順 6（デザイン指示書）の角丸・境界線・配色を優先。過剰なバッジや区切りの乱立を避け、コーポレートサイトとして落ち着いた仕上がりにする。

図解や表を表示する際は画像ではなく、コーディングで生成してください。

アイコンは基本的に Lucide React から使用すること。（SNS アイコンは Simple Icons を使用すること）
アイコンをプレースホルダ画像にするのは禁止。

画像は Unsplash から使用禁止。実際の画像は設置しないこと。

所在地を表示する場合は Googlemap を埋め込むので、Googlemap 表示エリアには絶対に画像を埋め込まないようにしてください。（プレースホルダーも置かないこと）
所在地ではなく、「対応エリア」の画像やイラストは生成禁止・配置禁止。
会社の所在地が明確に明記されている場合は、Googlemap を埋め込みピンで表示すること。

【技術要件】
必ず以下の技術要件で生成すること
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSS の @import ルール（Google Fonts など）は、必ず CSS ファイルまたは <style> タグの最上部に配置すること。他の CSS ルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。

【スタイリング要件】
- Next.js + Tailwind CSS へのリファクタリングを前提とした実装
- CSS 変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body 要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定
