**【手順.7-3】**

次に以下のサイト構成の**ユーザー向け情報**を欠落なく反映してページを作成してください。
以下のサイト構成は下層ページのサイト構成です。
プレビュー可能な形式で生成してください。
手順.7-2 の出力を踏まえて、TOP ページに以下の下層ページのサイト構成を反映した１ページのサイトとして出力してください。
TOP ページの内容はそのままで内容は変えたり省略しないでください。

【デザインの自主性】
- **ユーザー向けテキスト情報**は欠落なく反映する。**レイアウト・ビジュアルデザイン**は手順 6 とプロ判断で最適化してよい。構成案の【画像・配置】にピクセル単位で縛られない。

【原稿の厚み（必須）】
- **文章の要約・省略でページを薄くしない**。手順 7-2 で載せた文案に加え、下層の**全セクション・全箇条・全 FAQ**を欠落なく載せる。

【表示するテキストと「裏方メモ」の分離（必須）】
- 「【画像・配置の指示】」の本文、セクション見出しラベル（メインビジュアル、Why 等）、制作向け説明は**画面上に表示しない**。
- 表示してよいのは「【テキスト情報】」内の**見出し・本文・表・ボタン・FAQ**のみ。レイアウト指示の丸写し禁止。情報はレイアウトとスタイルに翻訳する。
- 画像枠は**短いプレースホルダ**のみ。長文の撮影指示はコメントまたは `aria-label` 要約。接頭辞「【画像プレースホルダー】」等をユーザーに見せない。
- ヒーロー等で指定された**キャッチ・タイトル**はオーバーレイで必ず前面表示。

【UI / デザイン品質（プロ品質）】
- 余白・`max-w`・グリッド揃えを統一。タイポの階層を明確に。hover / focus-visible。主 CTA の視認性。横スクロール禁止。手順 6 のトーンに沿った落ち着いた仕上がり。
- ヒーローに**画像（またはプレースホルダ）が必ずあること**（単色のみのヒーロー禁止）。H1・キャッチは**本文より大きく**、オーバーレイでコントラストを確保していること（手順 6 の「タイトル」px に従う）。

【表現・文字面（絵文字禁止・必須）】
- Unicode 絵文字を見出し・本文・ボタン・FAQ・ナビ・プレースホルダ・aria-label／alt に含めない。コミットメッセージ・ソース内のユーザー向け文言にも入れない。
- 構成案・サイト構成テキストに無い絵文字は追加しない（原稿にのみある場合はそのまま可）。この手順への応答にも絵文字を使わない。
- 強調はタイポ階層と Lucide 等の SVG アイコンで行う。

プレビューが可能な形式（App コンポーネントをデフォルトエクスポートとする単一ファイル）で出力してください。

最後にヘッダーとフッターのナビを確認して、ページ全体を通して CTA やリンクが機能しているのかも確認してください。
デスクトップ版ではハンバーガーメニューではなくヘッダーにナビゲーション項目を設置すること。

ナビゲーションメニューの必須ルール
・指定がない限り項目は日本語
・項目は、そのページが何の内容かがわかりやすい端的な文言
・CTA のページへの項目は通常のテキストではなく、他の CTA ボタンと同じボタンデザインにする
・項目には該当ページへの内部リンクが設置されていること

【技術要件】
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSS の @import ルール（Google Fonts など）は、必ず CSS ファイルまたは <style> タグの最上部に配置すること。他の CSS ルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。
- 全コンテンツを縦積みにするのではなく、useState を用いてナビゲーション操作により表示コンテンツ（TOP/Service/Recruit など）が切り替わる、擬似的なルーティング機能を実装してください。

【スタイリング要件】
- Next.js + Tailwind CSS へのリファクタリングを前提とした実装
- CSS 変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body 要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定
