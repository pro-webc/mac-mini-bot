**【手順.7-2】**

以下のサイト構成に含まれる**ユーザー向けの情報**を欠落・改悪なく反映した 1 ページのコーポレートサイトを作成してください。
手順 6 のデザイン指示書を**トークン・トーンの正本**とし、**レイアウト・セクションの組み立て・余白・グリッドはプロとして自律的に設計**してよい。構成案は**文案と情報の正本**とし、【画像・配置】は参考に留める（ワイヤどおりの再現に縛られない）。

【原稿の厚み（必須）】
- **デザイン調整で文章量を減らさない**。構成案【テキスト情報】の**全見出し・本文・箇条・表・FAQ**を欠落・要約なく UI に載せる。見出しだけのセクションにしない。

【表示するテキストと「裏方メモ」の分離（必須）】
- 「【画像・配置の指示】」の本文、セクション見出しラベル、制作向け説明は**公開ページ上に表示しない**。
- 表示してよいのは「【テキスト情報】」内の**見出し・本文・表・ボタン・FAQ**のみ。レイアウト指示の丸写し禁止。
- 画像枠は**短いプレースホルダ**のみ。長文の撮影指示は `{/* */}` または `aria-label` に 1 文要約。接頭辞「【画像プレースホルダー】」等をユーザーに見せない。
- ヒーロー上に載せる**キャッチ・タイトル・CTA**は前面オーバーレイで表示。

【UI / デザイン品質（プロ品質）】
- 余白・`max-w`・タイポ階層・hover / focus-visible・主 CTA・横スクロール禁止。手順 6 に沿った落ち着いた仕上がり。
- **メインビジュアル（ヒーロー）**: **ヒーローには必ず画像を用いる**（`next/image` またはプレースホルダ枠。単色ベタ・模様のみで画像エリアを省略しない）。H1・キャッチは**手順 6 の「タイトル」px**で**本文より明確に大きく**実装する。画像/プレースホルダの上は**暗オーバーレイ**または**背景専用グラデーション**で文字を読ませ、キャッチ/H1/サブコピー/主CTAの縦スタックに十分な `gap` を取る。主CTAはヒーロー内で最も視認性が高い塗りボタンにする（装飾の `shadow-*` は使わない）。
- **没入感・現代UI**: 手順6の**世界観**に沿い、セクション間で**背景・段組のリズム**を付け単調な縦積みを避ける。ホバー／フォーカスは短い `transition`。スクロール入場は `motion-safe:` 内の軽い演出に限り、`motion-reduce:` で抑制。

図解や表を表示する際は画像ではなく、コーディングで生成してください。

アイコンは基本的に Lucide React から使用すること。（SNS アイコンは Simple Icons を使用すること）
アイコンをプレースホルダ画像にするのは禁止。

Unsplash・外部ストック URL は禁止。**マニュアル段**ではビジュアル枠は `ImagePlaceholder` または**同一返答のフェンスで出力する SVG 等**とし、**リポジトリに無いファイルを `next/image` の `src` だけで参照しない**（404・表示欠落の典型）。最終リファクタで `public/images/` の実ファイルと正しい `src`（`/images/...`、`/public` を含まない）へ置換する前提で整合させること。

所在地を表示する場合は Googlemap を埋め込むこと。
所在地ではなく、「対応エリア」の画像やイラストは生成禁止・配置禁止。

・指定がない限り項目は日本語
・項目は、その項目が何のセクションかわかりやすい端的な文言
・CTA への項目は通常のテキストではなく、他の CTA ボタンと同じボタンデザインにする
・項目には該当セクションへの内部リンクが設置されていること

【技術要件】
必ず以下の技術要件で生成すること
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSS の @import ルール（Google Fonts など）は、必ず CSS ファイルまたは <style> タグの最上部に配置すること。他の CSS ルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。

【スタイリング要件】
- Next.js + Tailwind CSS へのリファクタリングを前提とした実装
- CSS 変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body 要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定

CTA は遷移できるか確認してください。
