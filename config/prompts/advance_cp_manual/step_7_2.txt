**【手順.7-2】**

以下のサイト構成に含まれる**ユーザー向けの情報**を欠落・改悪なく反映した TOP ページを作成してください。まずは TOP のみ作成してください。

【デザインの自主性（構成案との役割分担）】
- **構成案は情報・文案・導線の正本**。**レイアウト・ビジュアルは手順 6 とプロ判断で自律的に設計**してよい。【画像・配置の指示】は参考に留め、ワイヤの文言どおり再現に縛られない。手順 6 の配色・フォント範囲・禁止事項は遵守。

【原稿の厚み（必須）】
- **レイアウトのために文章を短くしない**。箇条・表・FAQ の**項目数・文量を要約・省略しない**。**見出しのみのセクションを作らず**、【テキスト情報】の**全文**を UI に載せる。

【表示するテキストと「裏方メモ」の分離（必須）】
- 「【画像・配置の指示】」の本文、セクション見出しラベル（メインビジュアル、Why 等）、制作向け説明は**公開ページ上に表示しない**。
- 表示してよいのは「【テキスト情報】」内の**見出し・リード・本文・表・ボタン・FAQ**のみ。レイアウト指示の丸写し禁止。情報はレイアウトに翻訳する。
- 画像枠は**短いプレースホルダ**のみ。長文の撮影指示は `{/* */}` または `aria-label` に 1 文要約。接頭辞「【画像プレースホルダー】」等をユーザーに見せない。
- 「画像の上に載せる」**キャッチ・タイトル・CTA**は必ず**前面オーバーレイ**で表示。

【UI / デザイン品質（プロ品質）】
- 余白・`max-w`・グリッド揃えの統一。タイポ階層。hover / focus-visible。主 CTA。横スクロール禁止。手順 6 に沿った落ち着いたコーポレート仕上がり。

図解や表を表示する際は画像ではなく、コーディングで生成してください。

アイコンは基本的に Lucide React から使用すること。（SNS アイコンは Simple Icons を使用すること）
アイコンをプレースホルダ画像にするのは禁止。

画像は Unsplash から使用禁止。実際の画像は設置しないこと。

所在地を表示する場合は Googlemap を埋め込むこと。
所在地ではなく、「対応エリア」の画像やイラストは生成禁止・配置禁止。

【技術要件】
必ず以下の技術要件で生成すること
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSS の @import ルール（Google Fonts など）は、必ず CSS ファイルまたは <style> タグの最上部に配置すること。他の CSS ルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。

【スタイリング要件】
- Next.js + Tailwind CSS へのリファクタリングを前提とした実装
- CSS 変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body 要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定
