[重要] TOP ページと手順.7-3 で生成した web サイトの内容は絶対に変えたり省略しないでください。（ページを追加するだけ）

【表示・品質（必須）】
- テキスト情報は省略しない。ただし**【画像・配置の指示】の長文**・セクションラベル・「【画像プレースホルダー】」接頭辞は**画面上から除去**し、画像枠は短いプレースホルダに統一。撮影指示はコメント / `aria-label` 要約。
- **デザインの自主性**: ワイヤに縛られず、手順 6 の範囲でレイアウト・余白・コンポーネント構成を**プロ品質になるよう自主調整**してよい（**文言・数値・CTA ラベルは変更禁止**）。
- **UI プロ品質**: 余白・`max-w`・見出し階層・CTA 一貫性・hover / focus-visible・横スクロールなし。手順 6 のデザイン規則に沿う。

プレビューが可能な形式（App コンポーネントをデフォルトエクスポートとする単一ファイル）で出力してください。

最後にヘッダーとフッターのナビを確認して、ページ全体を通して CTA やリンクが機能しているのかも確認してください。（遷移できるか確認すること）
デスクトップ版ではハンバーガーメニューではなくヘッダーにナビゲーション項目を設置すること。

ナビゲーションメニューの必須ルール
- 指定がない限り項目は日本語
- 項目は、そのページが何の内容かがわかりやすい端的な文言
- CTA のページへの項目は通常のテキストではなく、他の CTA ボタンと同じボタンデザインにする
- 項目には該当ページへの内部リンクが設置されていること
- この web サイトはページ数が多いのでナビゲーションを表示する際はサービスページなどをドロップダウンで表示すること

【技術要件】
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSS の @import ルール（Google Fonts など）は、必ず CSS ファイルまたは <style> タグの最上部に配置すること。他の CSS ルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。
- 全コンテンツを縦積みにするのではなく、useState を用いてナビゲーション操作により表示コンテンツ（TOP/Service/Recruit など）が切り替わる、擬似的なルーティング機能を実装してください。

【スタイリング要件】
- Next.js + Tailwind CSS へのリファクタリングを前提とした実装
- CSS 変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body 要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定
