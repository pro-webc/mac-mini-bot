【ナビゲーションメニューの必須ルール】
- 指定がない限り項目は日本語
- 項目は、そのページが何の内容かがわかりやすい端的な文言
- CTAのページへの項目は通常のテキストではなく、他のCTAボタンと同じボタンデザインにする
- 項目には該当ページへの内部リンクが設置されていること

【技術要件】
- Next.js (App Router), React, Tailwind CSS
- セクション単位でコンポーネントを作成せよ。
- CSSの@importルール（Google Fontsなど）は、必ずCSSファイルまたは<style>タグの最上部に配置すること。他のCSSルール（:root、セレクタ、プロパティなど）よりも前に記述する必要がある。これを守らないとビルドエラーが発生する。
- 全コンテンツを縦積みにするのではなく、useStateを用いてナビゲーション操作により表示コンテンツ（TOP/Service/Recruitなど）が切り替わる、擬似的なルーティング機能を実装してください。

【スタイリング要件】
- Next.js + Tailwind CSSへのリファクタリングを前提とした実装
- CSS変数を定義する場合、各要素に明示的にクラスやスタイルで色を指定
- body要素への色の継承に依存せず、テキスト要素ごとに色を指定
- ボタンの状態（active/inactive/hover）ごとに、明示的に色を指定
