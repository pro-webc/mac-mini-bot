# デザインルール（配色・レイアウト・UI 品質）

## マニュアル手順内（プラン別 .txt）

### BASIC LP

- **手順 5（`step_5.txt`）**: クライアントの色・雰囲気（`{{HP_COLOR_CLIENT}}` / `{{MOOD_CLIENT}}`）、参考 URL（`{{REFERENCE_URL_BLOCK}}`）を前提に配色・フォント・角丸・禁止事項などの**デザイン 3 点セット**を出力する流れ。
- **手順 6・7**: ワイヤとデザイン方針を統合した**デザイン指示書**（手順 7 の出力が手順 8-1 の入力 `{{STEP_7_OUTPUT}}`）。
- **手順 8-1**: デザイン指示書に基づき 1 枚 LP を生成（この段ではダミーテキスト可、ヘッダーフッター禁止）。
- **手順 8-2**: デザインを保ちつつ**サイト構成（step 4）を 100% 反映**。FAQ はアコーディオン必須。画像はプレースホルダで説明（Unsplash 禁止）。ヒーローには必ず画像プレースホルダまたは画像。
- **スタイリング要件（8-2 等）**: Tailwind 前提、CSS 変数使用時も要素ごとに色指定、body 継承に依存しない、ボタン状態ごとに色指定。

### BASIC-CP / STANDARD / ADVANCE（手順 7 系）

- **手順 7-1**: デザイン指示書ペースト＋技術要件の短い箇条書き＋絵文字禁止。
- **手順 7-2**: 手順 6 を**トークン・トーンの正本**、構成案を**文案の正本**とする二層構造。レイアウトはプロとして自律調整可だが**文章量を減らさない**。ヒーローは必ず画像エリア、H1/キャッチは手順 6 のサイズ感より大きく、オーバーレイで可読性確保。装飾の `shadow-*` 禁止。
- **表示とメモの分離**: 「【画像・配置の指示】」等は公開 UI に出さない、プレースホルダは短く、など。

## 共通技術ブロック（`technical_spec_prompt_block.txt`）

`config/prompts/common/technical_spec_prompt_block.txt` に、生成モデル向けの**デザイン系強制**がまとまっている（サイト出力の `TECH_REQUIREMENTS.md` と同内容）。

要点:

- **design_spec の color_scheme / tokens を最優先**。勝手な全面ベタ塗りや派手色の追加は禁止。
- `body`/`html` 背景は低彩度、ブランド色はヘッダー帯・CTA・アクセントに限定。
- テキスト色はパレット内で種類を抑え、コントラスト確保。
- ボタン・リンクの hover/active/focus-visible/disabled を primary 系の明度差で統一（虹色複数 CTA 禁止）。
- タイポは仕様の typography に沿い、h1 はページ 1 つ、長文は可読幅。
- セクション間の縦リズム（`py` スケール）を揃える。
- `bg-[#..]` 任意値は design_spec の色の反映に限定。

## 仕様 dict の `design_spec`

パース後の YAML に `color_scheme`・`typography`・`layout_mood`・`design_principles` が入り、**サイト台本のメタ**として保持されます（`modules/llm/basic_lp_spec.py` のデフォルト例参照）。

## Manus リファクタ（`refactoring_instruction_handwork.txt`）

- デザイン・レイアウト・配色・テキストは **Canvas と同一維持**（文言・数値・手順 6 のカラーコードは変更禁止）。
- ただしコンポーネント分割に伴いグリッド・余白・レスポンシブは**プロ品質になるよう調整可**。
- Wire メタの除去のみ可。ユーザー向け原稿の要約・統合は禁止。
